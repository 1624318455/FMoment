# FMoment 评估体系

## 评估分类

### 行为评估
检查约束遵守情况：
- 代码分层是否正确
- 禁令是否被违反
- 资源使用是否在阈值内

### 功能评估
任务完成质量：
- 截图功能是否正常
- 取色精度是否准确
- 配色算法是否正确
- 色卡管理是否完整

## 评估场景

### 正常场景
1. 截图 → 标注 → 保存
2. 取色 → 复制 → 加入色卡
3. 创建色卡 → 添加颜色 → 导出
4. 生成渐变 → 复制 CSS
5. 检查对比度 → 获取建议

### 边界场景
1. 超大尺寸截图（4K/8K）
2. 极端颜色值（纯黑/纯白）
3. 大量颜色项（100+）
4. 快速连续操作
5. 多窗口同时操作

### 错误场景
1. 磁盘空间不足
2. 数据库损坏
3. 网络连接失败
4. 权限被拒绝
5. 插件初始化失败

### 安全场景
1. 路径遍历攻击
2. SQL 注入
3. 资源耗尽攻击
4. 敏感信息泄露

## 评估指标

### 行为评估得分
- 代码分层正确率 ≥ 95%
- 禁令违反次数 = 0
- 资源使用超标次数 ≤ 2

### 功能评估得分
- 截图成功率 ≥ 99%
- 取色精度误差 ≤ 1%
- 配色算法正确率 = 100%
- 色卡管理完整性 ≥ 99%

## 评估流程

### 快速检查（每次 PR）
1. 静态分析
2. 单元测试
3. 集成测试
4. 行为评估

### 全量检查（每晚）
1. 全部测试用例
2. 性能测试
3. 安全扫描
4. 代码覆盖率

## 门禁规则

### 行为评估
- 得分 < 90% → 阻止合并
- 得分 90-95% → 警告
- 得分 ≥ 95% → 通过

### 功能评估
- 得分 < 80% → 阻止合并
- 得分 80-90% → 警告，需 issue 跟踪
- 得分 ≥ 90% → 通过

## 评估工具

### 自动化工具
- `flutter analyze` - 静态分析
- `flutter test` - 单元测试
- `dart format` - 格式检查

### 手动检查
- 代码审查
- 架构评审
- 安全审计
